# 개발 기초 지식 정리 보고서

## 1. Git이란 무엇이며, GitHub와 어떤 차이가 있는가?

### Git (Version Control System)
* **정의:** 로컬에서 소스 코드의 변경 이력을 관리하는 **분산 버전 관리 도구**입니다.
* **역할:** 코드의 수정 사항을 기록(Commit)하고, 과거 시점으로 되돌리거나(Checkout), 가지를 쳐서(Branch) 독립적인 작업을 가능하게 합니다.
* **특징:** 인터넷 연결 없이도 내 컴퓨터(로컬)에서 모든 작업이 가능합니다.

### GitHub (Cloud Service)
* **정의:** Git으로 관리하는 프로젝트를 클라우드에 올릴 수 있는 **원격 저장소 호스팅 서비스**입니다.
* **역할:** 전 세계 개발자들과 코드를 공유하고, 협업(Pull Request), 이슈 관리, 오픈소스 기여 등을 할 수 있는 플랫폼 역할을 합니다.
* **특징:** 협업을 위한 소셜 네트워크 기능이 강하며, 클라우드 기반이므로 인터넷 연결이 필요합니다.

> **핵심 차이:** Git은 내 컴퓨터 안의 **관리 도구**이고, GitHub는 그 결과물을 올리는 **온라인 보관소 및 커뮤니티**입니다.

## 2. 개발자가 마크다운(Markdown)을 사용해야 하는 이유

### 마크다운(.md) 사용 이유
1.  **가독성과 생산성:** 별도의 문서 도구(MS Word 등) 없이 텍스트만으로 제목, 목록, 표 등을 빠르게 구성할 수 있습니다.
2.  **버전 관리 최적화:** 순수 텍스트 형식이므로 Git을 통해 변경된 줄(Line) 단위의 차이점을 명확히 확인할 수 있습니다.
3.  **플랫폼 호환성:** GitHub, Notion, Slack, 기술 블로그 등 대부분의 개발 생태계에서 표준 문서 포맷으로 지원합니다.

### 주로 쓰이는 곳
* **README.md:** 프로젝트의 대문이자 가이드라인 파일.
* **PR/Issue:** GitHub에서 기능 구현 설명이나 버그 리포트를 작성할 때.
* **기술 블로그/위키:** 개발 지식을 기록하고 공유하는 기술 문서 사이트.

## 4. 라이브러리, 프레임워크, API의 개념과 차이점

### 개념 정의
* **라이브러리(Library):** 특정 기능을 수행하기 위해 미리 작성된 **코드 조각(함수/클래스)의 집합**입니다. (예: Python의 Pandas, JS의 Lodash)
* **프레임워크(Framework):** 애플리케이션 개발의 **기본 구조와 규칙을 제공하는 틀**입니다. (예: Spring, Django, React)
* **API(Application Programming Interface):** 소프트웨어 간에 데이터를 주고받기 위한 **소통 규칙 및 창구**입니다. (예: Google Map API, 날씨 정보 API)

### 핵심 차이점 (제어의 역전)
* **주도권(Inversion of Control):**
    * **라이브러리:** 개발자가 필요한 시점에 **직접** 호출하여 사용합니다. (내가 주체)
    * **프레임워크:** 프레임워크가 정의한 흐름 내에 개발자가 코드를 **끼워 넣습니다.** (프레임워크가 주체)
* **API:** 위 도구들을 사용하기 위해, 혹은 외부 서비스와 통신하기 위해 지켜야 할 **약속**입니다.
